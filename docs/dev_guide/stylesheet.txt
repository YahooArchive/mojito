================================
Mojito Documentation Style Guide
================================

Word Usage
##########
- Action Context / ActionContext / ac (actionContext) - concept/module or class/object
- addon - not "add on" or "add-on" 
- apps/applications - use 'application' for formal documentation, but for FAQs, code examples, or tables (small spaces), 'app' is acceptable.
- auto-generated - not "autogenerated" or "auto generated" 
- basename
- Cocktails  - not cocktails
- codebase - not "code-base" or "code base"
- evented
- file system - not "filesystem"
- formatter
- inline(d) - used for assets, such as 'inline CSS' or 'inlined CSS'. 
- hardcoding - not "hard-coding" or "hard coding"
- Manhattan (not "manhattan")
- metadata - not "meta data"
- middleware - not "middle-ware" or "middle ware"
- mojit - not Mojit
- Mojito - not "mojito" or "Mojito framework"
- Mustache tags - not "mustache" tags
- mutator
- namespace - not "name space"
- Node.js - not "Node" or "NodeJS"
- personalization   
- prepending - not "pre-pending" 
- property - not "field" when referring to objects
- proxied
- quickstart - not 'quick start'
- refire - not "re-fire"
- repurpose - not "re-purpose" 
- roadmap - not "road-map" or "road map"
- rollup (n.) - not "roll-up" or "roll up"
- runtime - not "run-time" or "run time"
- scalability
- screenshot - not "screen shot"
- scrollable
- screencast
- slidecast
- startup (n.) - not "start-up" or "start up"
- subdirectory - not "sub-directory"
- templating
- unenhanced
- unescaped
- view - refers to the rendered template file.
- view template - refers to the template file used for Mojito views. 
- whitespace
- wildcard

Capitalization
##############

- The name of addons should be uppercase and in monospace font ``Assets`` addon.
- Cocktails, Mojito, and Manhattan should be capitalized.

Conventions
###########

- Use 'objects' for configuration and not 'sections'. Adding new terminology is confusing.
- Variable substitutes are given in curly braces. For example: {mojit_directory}/assets 
- Command-line prompt should use the following format and use literal tags: $ ls

Formatting
##########

- GUI components are in bold.
- All commands, mojit, and application names should be in monospace font.
- Module and mojit names should also be in monospace font.

Code
####

- JSON
   - Data Types: string, boolean, array, object, number (based on json.org)
   - Use 'fields' for keys in key-value pairs (based on Wikipedia entry for JSON)
   - Functions:
      - use for controller functions, key-value names pointing to functions such as "init": function() {}.
      - do not use parentheses for functions in text. For example: "use ac.done", NOT "use ac.done()"
   - Methods:
      - use for Mojito API methods as they are called from objects or classes.
   - Properties should be in double quotes: "key": "value"
- Classes/Objects: 
   - MojitProxy/mojitProxy 
   - ActionContext/actionContext
